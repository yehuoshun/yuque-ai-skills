# yuque_batch_get_docs — 批量获取文档详情

## 端点

并发 `GET /api/v2/repos/:book_id/docs/:id`（无原生批量端点，MCP 层 `Promise.all` 实现）

## 用途

一次请求获取多个文档的完整内容（body/body_html/body_lake），避免串行调用。

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `book_id` | string | ✅ | 知识库 ID 或 namespace（所有文档共享同一知识库） |
| `ids` | string | ✅ | 文档 ID 的 JSON 数组，如 `[123, 456]` 或 `["slug-a", "slug-b"]`，最多 20 个 |
| `raw` | boolean | ❌ | 返回原始完整 JSON（默认 false，返回精简字段） |

## 返回

```json
{
  "total": 3,
  "ok": 2,
  "docs": {
    "123": { "id": 123, "title": "...", "body": "..." },
    "456": { "id": 456, "title": "...", "body": "..." }
  },
  "errors": [
    { "id": 789, "detail": { ... } }
  ]
}
```

- `total`: 请求的文档总数
- `ok`: 成功获取的数量
- `docs`: 按 id 分组的文档详情
- `errors`: 获取失败的文档及错误详情

## 约束

- **只读操作**，不修改任何数据
- 最多 20 个文档（超过返回错误）
- 所有文档必须在**同一知识库**内
- 并发请求，不保证返回顺序（但结果按 id 分组）

## 示例

```json
{
  "book_id": "yehuoshun/my-repo",
  "ids": "[273639914, 269401978, 269401944]"
}
```

## 注意事项

- 语雀 API 有频率限制（默认 1000 次/小时），批量操作会消耗对应数量的配额
- 大量文档建议分页调用，每次 ≤ 20 个
- 跨知识库的文档需多次调用（每次指定不同 book_id）
