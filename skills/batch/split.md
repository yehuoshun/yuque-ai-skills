# 文档拆分（split）

按标题层级自动切割长文档为多篇独立文档，保留格式，自动建目录。

## 触发词

```
「拆分」「拆成多篇」「按章节拆分」「按标题切割」
「这篇太长了，拆一下」「按 H2 拆分」
```

---

## 拆分策略

### 按标题切割

```
默认：按 H2（##）切割

可选：
  H1（#）→ 每章一篇（适合书/系列教程）
  H2（##）→ 每节一篇（默认）
  H3（###）→ 每个小节一篇（超细粒度）

用户指定：「按章节拆」「按 H3 拆」
```

### 切割逻辑

```
1. yuque_get_doc 读 body
2. 按指定标题级别（如 H2）定位切割点
3. 两切割点之间的内容 → 一个子文档
4. 子文档标题 = 切割点的 H2 标题文字
5. 原 H1 标题作为系列前缀（可选）：
   「Java教程 — 第一章 基础概念」
6. 第一个切割点之前的内容（如有）→ 归为「前言」文档

特殊处理：
  - 代码块内的 ## 不算切割点（在 ``` 包裹内）
  - 表格内的 | 和 - 不管，只认行首的 #
  - 无 H2 的纯文本 → 提示「未找到可切割的标题层级」
```

### 保留内容

```
每个子文档继承：
  ✓ 对应标题 + 内容正文
  ✓ 父级标题作为上下文（如 H2 所属的 H1）
  ✓ 图片、表格、代码块完整保留
  ✓ 原文档底部的公共内容（如「参考链接」「关于作者」）——可选保留
  ✗ 不计原文档开篇的导航/目录（纯 TOC 类内容）
```

---

## 操作流程

```
1. 定位源文档（同其他 skill 的文档定位逻辑）
2. yuque_get_doc → 读 body
3. 预览切割方案：

📋 切割预览：《{title}》（共 {N} 字）

目标层级：{H1/H2/H3}
系列前缀：{默认原标题 / 用户指定 / 无}

切割为 {M} 篇：

  #  标题            字数
  1  前言             120
  2  第一章 基础       1860
  3  第二章 进阶       772
  ...

操作：
  「确认拆分」
  「改层级 H3」
  「排除 序号N」
  「改前缀 {新前缀}」
  「合并 {序号X+Y}」
  「取消」

4. 用户确认 → 逐篇 yuque_create_doc
5. 自动建 TOC：yuque_update_toc 将子文档挂到目录
6. 可选：yuque_delete_doc 删除原长文（⚠️ 需二次确认）
```

---

## 标题命名规则

```
默认：子文档标题 = H2 标题文字

如果子文档在前 3 级标题中已有相同的标识（如「1.」「第一章」），保留序号：

  原文 H2: ## 1. Docker 安装
  子文档标题: 1. Docker 安装

系列前缀：
  用户指定「前缀设为《Docker教程》」
  → 子文档标题: Docker教程 — 1. Docker 安装

  未指定 → 无前缀

父标题上下文：
  可选在每个子文档开头加引用行：
  > 本文来自《{原标题}》，拆分为 {N} 篇独立文档
  
  默认不加（保持子文档干净），用户要求再加。
```

{如已删原文档：}

```

## 上限保护

| 项 | 上限 |
|----|------|
| 子文档数 | 30 篇 |
| 单篇最小字数 | 50 字（少于则提示合并到相邻） |
| 并发创建 | 3 篇 |

---

## 错误处理

| 场景 | 处理 |
|------|------|
| 文档未找到 | 「未找到文档」 |
| 无 H2/H3 标题 | 「未找到 {层级} 标题，无法切割。试试改层级」 |
| 子文档数 > 30 | 「共 {n} 篇，上限 30 篇，建议用更粗的层级切割」 |
| 子文档 < 50 字 | 提示并建议合并到相邻子文档 |
| 创建失败 | 单篇失败不影响其他 |
| 删除确认 | ⚠️ 二次确认「原文档将被硬删除，确认？」 |
