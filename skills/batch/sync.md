# 文档镜像 & 知识库同步（sync）

文档级镜像（改一处全更新）+ 知识库级同步（两个库保持镜像）。二合一。

## 触发词

```
文档级：「镜像到XX库」「同步副本」「更新所有镜像」「这篇在哪还有」
知识库级：「同步两个库」「两个库保持一致」「镜像知识库」「看看两个库差多少」
```

## 场景判定

```
含「知识库/两个库/全库/库之间」 → 知识库同步
含「镜像这篇/同步副本/更新镜像/这篇」 → 文档镜像
只给知识库名 → 知识库同步
只给文档名 → 文档镜像
```

# 场景一：📄 文档镜像

## 三个操作

| 操作 | 行为 | 触发 |
|------|------|------|
| 🔗 建立镜像 | 复制到目标库 + 记录镜像关系 | 「镜像到XX库」 |
| 🔄 同步更新 | 源改了 → 推送到所有镜像 | 「同步镜像」「更新所有副本」 |
| 🔍 查镜像 | 列出这篇文档的所有镜像副本 | 「查看镜像」 |

## 镜像关系存储

```
存在知识库的图谱文档（_知识图谱_{库名}）中，追加镜像表：

## 🪞 文档镜像
| 源文档 | 源库 | 镜像库 | 镜像 doc_id | 最后同步 |
|--------|------|--------|------------|---------|
| Docker入门 | 个人笔记 | 团队共享 | 456 | 2026-05-23 |
| Docker入门 | 个人笔记 | 公开博客 | 789 | 2026-05-20 |
```

## 建立镜像

```
1. 用户：「把《Docker入门》镜像到团队共享库」
2. yuque_get_doc 读源文档
3. yuque_create_doc 在目标库创建副本
4. 检查图谱文档是否存在 → 不存在则创建 → 追加镜像表行
5. 返回：✅ 镜像已建立 → {目标库链接}
```

## 同步镜像

```
1. 用户：「同步《Docker入门》的镜像」或「更新所有镜像」
2. 读图谱文档镜像表 → 拿到全部镜像 (book_id, doc_id)
3. yuque_get_doc(源库, 源doc_id) 读最新 body
4. 逐镜像 yuque_update_doc(镜像库, 镜像doc_id, body=最新)
5. 更新 synced_at
6. 输出同步报告
```

### 同步报告

```
✅ 镜像同步完成：《{title}》

源：{源库名} — 最后修改 {updated_at}
镜像：{N} 个

  1. {镜像库名} → 已更新 ✓
  2. {镜像库名} → 已更新 ✓

上次同步：{时间} → 本次：{时间}
```

## 查镜像

```
「这篇在哪还有镜像？」

→ 读图谱文档镜像表 → 输出：

🪞 《{title}》镜像位置：
  1. {库名} — doc_id={id} — 上次同步 {时间}
  2. {库名} — doc_id={id} — 上次同步 {时间}
```

# 场景二：📚 知识库同步

## 四种模式

| 模式 | 行为 | 触发 |
|------|------|------|
| ➡️ 单向推送 | A 的变更 → 推到 B | 「从A同步到B」 |
| ⬅️ 单向拉取 | B 的变更 → 拉到 A | 「从B拉取到A」 |
| 🔄 双向同步 | A↔B 互相镜像 | 「双向同步A和B」 |
| 🔍 差异检测 | 只看不执行 | 「看看差多少」「对比」 |

## 差异检测

```
逐文档按 title 匹配同名文档：

⚡ 流程：
  1. yuque_list_docs(A) → 源库全部文档（title/id/updated_at）
  2. yuque_list_docs(B) → 目标库全部文档
  3. 逐对匹配：
     - title 精确匹配 → 同名文档
     - 同名文档 updated_at 对比 → 判断谁更新
     - title 未匹配 → 标记「仅一侧存在」

📊 差异结果：
  源库独有 → ➕ 待在目标库创建
  目标库独有 → ➖ 仅目标库有（同步方向决定处理）
  两边都有、源更新 → ✏️ 待更新目标库
  两边都有、目标更新 → ⏸️ 目标库更新（推送模式跳过，双向模式反向同步）
  两边相同 → ✅ 一致跳过
```

## 预览输出

```
🔍 同步预览：《{A}》↔《{B}》
模式：{单向推送/双向同步} | 上限 100 篇

📊 差异统计：
  源库 {Na} 篇 | 目标库 {Nb} 篇

  ➕ 待创建：{n} 篇
  ✏️ 待更新：{m} 篇
  ➖ 仅目标库有：{k} 篇
  ✅ 一致跳过：{s} 篇

📋 详情（前 20 条）：
  ➕ 《文档1》→ 目标库创建
  ✏️ 《文档2》(源 05-20 > 目标 03-01) → 更新目标库
  ...

{双向同步冲突：}
⚠️ 冲突检测：
  《文档4》两边都改了 — 源 05-20 / 目标 05-22 → 按策略处理

---
操作：
  「确认同步」
  「排除 文档X」
  「改模式 {推送/拉取/双向}」
  「取消」
```

## 冲突处理

```
双向同步，两边同时改了同一篇：

  选最新 → 自动用 updated_at 更大的一方覆盖（默认）
  选A优先 → A 是权威源，A的变更覆盖 B
  选B优先 → B 是权威源，B的变更覆盖 A

首次提示：「检测到 {N} 处冲突，默认选最新处理。是否同意？」
用户同意 → 自动处理
用户不同意 → 逐条手动选择
```

## 执行同步

```
1. 创建类：yuque_get_doc(A) → yuque_create_doc(B)
2. 更新类：yuque_get_doc(A) → yuque_update_doc(B)
3. 并发 3 篇

4. 输出同步报告：

✅ 同步完成 — {A} → {B}
📊 {N} 篇 | ➕创建 {C} | ✏️更新 {U} | ⏭️跳过 {S} | ❌失败 {F}

➕ 已创建：
  《{title}》→ {链接}

✏️ 已更新：
  《{title}》({旧时间} → {新时间}) → {链接}

{有冲突：}
⚠️ 冲突（已按{策略}处理）：
  《{title}》— 用{源/目标/最新}覆盖

{有失败：}
❌ 失败：
  《{title}》— {原因}
```

## 增量同步

```
首次全量同步后记录时间戳。

存储：源库图谱文档追加：
  | 同步 | {A} → {B} | 2026-05-23 18:00 | {N}篇 |

下次增量：
  「增量同步A到B」
  → yuque_list_docs(A) 筛选 updated_at > 上次同步时间
  → 只处理变更文档
  → 更新同步时间戳
```

## 上限保护

| 场景 | 上限 |
|------|------|
| 文档镜像 | 1 篇 → 最多 5 个镜像 |
| 知识库同步 | 100 篇/次，超出提示分批 |
| 预览详情 | 前 20 条 + 统计摘要 |

---

## 错误处理

| 场景 | 处理 |
|------|------|
| 源库/目标库未找到 | 「未找到知识库，检查名称后重试」 |
| 源文档未找到 | 「未找到文档」 |
| 目标库无权限写入 | 「目标库无写入权限」 |
| 镜像目标库已有同名文档 | 「目标库已存在《{title}》，是否覆盖？跳过高亮同名冲突」 |
| 图谱文档不存在 | 自动创建 |
| API 限流(429) | 等 Retry-After，最多重试 3 次 |
