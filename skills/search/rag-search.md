# RAG 问答模式

基于语雀知识库的检索增强生成（RAG）问答。

## 什么时候用

- 用户问具体问题：「美菱有多大」「怎么刷初始」「PT 怎么算」
- 需要从文档中提取答案，而不是只返回文档列表

## 流程

```
用户提问
    ↓
1. HyDE 搜索 → yuque_hyde_search（LLM 生成关键词 + 多路搜索）
    ↓
2. 获取文档内容 → yuque_get_doc（取前 3-5 篇）
    ↓
3. LLM 阅读理解 → 总结答案 + 引用来源
    ↓
4. 返回答案
```

## 调用步骤

### Step 1: 生成搜索关键词

用 HyDE Prompt 生成 5-10 个关键词：

```
你是一个知识库搜索助手。根据用户提问，生成 5-10 个搜索关键词用于语雀全文搜索。

## 关键词要求
- 中文优先，其次是英文单词
- 不要包含连字符(-)、点号(.)、下划线(_)
- 覆盖同义词、缩写展开、中英文对照
- 技术术语优先用中文表达

## 输出格式
只输出逗号分隔的关键词：
关键词1,关键词2,关键词3,...
```

### Step 2: 调用搜索

```
yuque_hyde_search(
  keywords="美菱,角色介绍,角色资料,美菱年龄,美菱身高",
  scope="your-group",
  max_results=10
)
```

### Step 3: 获取文档内容

对搜索结果的前 3-5 篇文档：

```
yuque_get_doc(doc_id=12345)
yuque_get_doc(doc_id=12346)
yuque_get_doc(doc_id=12347)
```

### Step 4: LLM 总结答案

```
你是一个知识库问答助手。根据以下文档内容回答用户问题。

## 用户问题
{用户问题}

## 相关文档
{文档1标题}
{文档1内容}

{文档2标题}
{文档2内容}

{文档3标题}
{文档3内容}

## 回答要求
1. 直接回答问题，不要泛泛而谈
2. 如果文档中没有答案，明确说"文档中未找到相关信息"
3. 引用来源格式：根据《{文档标题}》，...
4. 如果多个文档有冲突，指出冲突并给出最可能的答案
```

## 返回格式

```json
{
  "question": "美菱有多大",
  "answer": "根据《角色图鉴》文档，美菱是卡厄思梦境中的角色，年龄约 18 岁，身高 165cm...",
  "sources": [
    {
      "title": "角色图鉴",
      "url": "https://www.yuque.com/xxx/xxx",
      "relevance": "high"
    }
  ],
  "confidence": "high"
}
```

## 降级链

```
1. HyDE 搜索 → 有结果？→ 继续
       ↓ 无结果
2. 原始问题直接搜 yuque_search → 有结果？→ 继续
       ↓ 无结果
3. 返回"未找到相关文档"
```

## 注意事项

- 不要一次性获取太多文档（3-5 篇足够，避免 token 爆炸）
- 文档内容太长时，只传前 2000 字
- 如果搜索结果质量差，可以换关键词重试
- 如果文档内容不相关，直接告诉用户"文档中没有答案"，不要瞎编
