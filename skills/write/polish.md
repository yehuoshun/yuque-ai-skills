# AI 写作工作室（writing-studio）

分析写作风格 → 打磨粗糙笔记 → 风格迁移改写 → 模板填充输出。合官方 note-refine + style-extract 为一体，加风格迁移和模板写作。

## 触发词

```
风格分析：「分析一下这篇的写作风格」「我的文风是什么」「风格画像」
笔记打磨：「打磨一下」「润色」「整理成文章」「扩写」「把笔记转成文档」
风格迁移：「用{xx}的风格改写」「模仿{xx}的文风」「参照{xx}的写法」
模板写作：「按技术方案模板写」「生成周报」「写个复盘报告」
```

## 模式判定

```
含「风格/文风/画像/分析」且无「改写/迁移/模仿」 → 🔍 风格分析
含「打磨/润色/整理/扩写/转成/N次打磨」 → ✨ 笔记打磨
含「模仿/迁移/用..风格/参照..写法」 → 🎭 风格迁移
含「模板/周报/复盘/方案/纪要/PRD」 → 📋 模板写作
含「批量/全部小记」 → 批量模式（笔记打磨/N次打磨模式）
```

## 模式一：🔍 风格分析

### 输入

```
单篇模式：
  文档名/doc_id → yuque_get_doc 读 body → LLM 分析

对比模式：
  「分析这两篇的风格差异」→ 读两篇 → 对比分析

知识库模式：
  「分析{知识库}的写作风格」→ list_docs → 抽 N 篇 → 聚合分析 → 输出整体风格画像
```

### 风格画像维度（10 维）

```
{
  "tone": "正式/口语/幽默/严肃/讽刺/温情",
  "sentence_length": "短句为主/中长句/长句为主/混合",
  "vocabulary_level": "通俗/专业/学术/混杂",
  "structure_preference": "总分总/递进/并列/散点/叙事",
  "signature_patterns": ["爱用设问开头", "每段≤3句", "大量类比"],
  "paragraph_density": "稀疏(>5句/段)/适中(3-5句)/密集(≤3句)",
  "punctuation_traits": "多用破折号/爱用省略号/感叹号频繁/严谨规范",
  "quote_frequency": "频繁引用/偶尔引用/从不引用",
  "example_density": "每段举例/关键处举例/很少举例",
  "emotional_level": "冷静客观/温和克制/激情外露/讽刺调侃"

## 模式二：✨ 笔记打磨

三级打磨，按输入质量自动匹配，用户可指定级别。

### 三级定义

| 级别 | 输入特征 | 动作 | 触发词 |
|------|---------|------|--------|
| 轻度 | 有完整段落，缺润色 | 修正语病 + 统一术语 + 优化排版 + 修正标点 | 「轻度打磨」「润色」「N 次打磨」 |
| 中度 | 有要点但缺逻辑结构 | 重组结构 + 补全论证 + 添加过渡段 + 提炼标题 | 「整理」「中度打磨」 |
| 深度 | 碎片/语音转文字/乱序 | 提取主题 → 重建逻辑链 → 补全缺失内容 → 润色 | 「深度打磨」「整理成文章」「扩写」 |

### 自动匹配规则

```
word_count < 50 或 无标点/无标题 → 深度
word_count 50-500 或 有标题但无分段 → 中度
word_count > 500 且有分段 → 轻度
用户指定优先于自动匹配
```

### 输入来源

```
指定文档名/doc_id → yuque_get_doc 读 body → 打磨
指定小记 → yuque_get_note 读 → 打磨
直接粘贴文本 → 直接送入 LLM
「最近 N 条小记」→ yuque_list_notes → 逐条打磨

批量模式（「把小记都打磨成文档」）：
  yuque_list_notes → 逐条深度打磨 → yuque_create_doc 写入目标库
```

### N 次打磨

用户说「再打磨一次」「N 次打磨」→ 在当前输出基础上再跑一次，每次只改一个有改进空间的地方。

```
用户「N次打磨」→ 对上次输出：
  1. 找 1 处最可改进的地方
  2. 针对性修改
  3. 输出新版本，标注「{N}次打磨」
  上限 5 次
```

### Prompt 模板

```
任务：{轻度/中度/深度}打磨以下{笔记/文档/小记}。

{轻度要求：
- 修正语病、错别字、标点错误
- 统一术语（同一概念用同一个词）
- 优化排版（标题层级、列表格式、空行节奏）
- 不改变原文结构和信息
- 在末尾列出修改清单}

{中度要求：
- 分析现有要点，重组为合理结构
- 为每个主要论点补全论证或举例
- 添加过渡句，增强段落衔接
- 提炼各层级标题
- 在末尾列出结构调整说明}

{深度要求：
- 从碎片中提取核心主题
- 重建完整逻辑链（问题→分析→结论）
- 补全缺失的背景、论证、案例
- 添加引导和总结段落
- 在末尾列出重建说明}

原文：
{body}

输出格式：
## {推断或保留原标题}
{打磨后正文}

---
📝 修改说明：
{修改清单}
```

### 输出方式

```
直接返回 → 对话中输出完整打磨后文章
创建文档 → yuque_create_doc(
    book_id={指定或默认库},
    title="{从内容推断的标题}",
    body={打磨后正文},
    format="markdown"
  ) → 返回链接
覆盖原文 → ⚠️ 需确认后 yuque_update_doc
```

## 模式三：🎭 风格迁移

用参考文档的风格重写目标文档，保留事实内容，只换表达方式。

### 输入

```
目标文档（被改写的内容）+ 风格参考文档（模仿其风格）

指定方式：
  「用《{风格参考}》的风格改写《{目标文档}》」
  「把这段笔记写成{某文档}那样的风格」
  「模仿这个知识库的风格」
```

### 流程

```
1. yuque_get_doc 读风格参考文档 → 提取风格画像（同模式一的 10 维分析）
2. yuque_get_doc 读目标文档 → 提取核心事实和观点
3. LLM 用风格画像的规则重写目标文档的核心事实
4. 输出改写结果 + 风格对照说明
```

### 约束

```
- 不改变目标文档的事实信息和核心观点
- 不添加目标文档没有的内容
- 风格参考只影响表达方式，不影响内容
- 如风格参考和目标文档类型差异过大（如诗歌→技术文档），提示并降级为轻度风格化
```

### 输出格式

```
## 🎭 风格迁移

**目标文档**：《{A}》
**风格参考**：《{B}》
**风格匹配度**：{高/中/低}

---
{改写后正文}

---
📊 风格对照：
| 维度 | 原文 | 迁移后 |
|------|------|--------|
| 语气 | ... | ... |
| 句式 | ... | ... |
```

```

  1. yuque_list_notes → 获取小记列表

## 错误处理

| 场景 | 处理 |
|------|------|
| 源文档未找到 | 「未找到文档，检查名称后重试」 |
| 小记为空 | 「没有小记可打磨」 |
| 风格参考文档未找到 | 「参考文档未找到，将使用通用正式风格」 |
| 模板类型不识别 | 「不支持的模板类型，可选：技术方案/周报/复盘报告/会议纪要/API文档/PRD」 |
| 大纲过于简略 | 「信息不足，请补充：{缺失的关键信息列表}」 |
| 读取失败(429) | 等 Retry-After，最多重试 3 次 |
| 批量超过 20 条 | 「共 {n} 条，首批处理 20 条」 |
