# 笔记打磨（refine）

将粗糙笔记/小记/碎片打磨为结构化文章。三级打磨，按输入质量自动匹配。

## 触发词

```
「打磨一下」「润色」「整理成文章」「扩写」「把笔记转成文档」
「N 次打磨」「再打磨一次」
「批量打磨」「把小记都打磨成文档」
```

---

## 三级定义

| 级别 | 输入特征 | 动作 | 触发词 |
|------|---------|------|--------|
| 轻度 | 有完整段落，缺润色 | 修正语病 + 统一术语 + 优化排版 + 修正标点 | 「轻度打磨」「润色」「N 次打磨」 |
| 中度 | 有要点但缺逻辑结构 | 重组结构 + 补全论证 + 添加过渡段 + 提炼标题 | 「整理」「中度打磨」 |
| 深度 | 碎片/语音转文字/乱序 | 提取主题 → 重建逻辑链 → 补全缺失内容 → 润色 | 「深度打磨」「整理成文章」「扩写」 |

### 自动匹配规则

```
word_count < 50 或 无标点/无标题 → 深度
word_count 50-500 或 有标题但无分段 → 中度
word_count > 500 且有分段 → 轻度
用户指定优先于自动匹配
```

---

## 输入来源

```
指定文档名/doc_id → yuque_get_doc 读 body → 打磨
指定小记 → yuque_get_note 读 → 打磨
直接粘贴文本 → 直接送入 LLM
「最近 N 条小记」→ yuque_list_notes → 逐条打磨

批量模式（「把小记都打磨成文档」）：
  yuque_list_notes → 逐条深度打磨 → yuque_create_doc 写入目标库
```

---

## N 次打磨

用户说「再打磨一次」「N 次打磨」→ 在当前输出基础上再跑一次，每次只改一个有改进空间的地方。

```
用户「N次打磨」→ 对上次输出：
  1. 找 1 处最可改进的地方
  2. 针对性修改
  3. 输出新版本，标注「{N}次打磨」
  上限 5 次
```

---

## Prompt 模板

```
任务：{轻度/中度/深度}打磨以下{笔记/文档/小记}。

{轻度要求：
- 修正语病、错别字、标点错误
- 统一术语（同一概念用同一个词）
- 优化排版（标题层级、列表格式、空行节奏）
- 不改变原文结构和信息
- 在末尾列出修改清单}

{中度要求：
- 分析现有要点，重组为合理结构
- 为每个主要论点补全论证或举例
- 添加过渡句，增强段落衔接
- 提炼各层级标题
- 在末尾列出结构调整说明}

{深度要求：
- 从碎片中提取核心主题
- 重建完整逻辑链（问题→分析→结论）
- 补全缺失的背景、论证、案例
- 添加引导和总结段落
- 在末尾列出重建说明}

原文：
{body}

输出格式：
## {推断或保留原标题}
{打磨后正文}

---
📝 修改说明：
{修改清单}
```

---

## 输出方式

```
直接返回 → 对话中输出完整打磨后文章
创建文档 → yuque_create_doc(
    book_id={指定或默认库},
    title="{从内容推断的标题}",
    body={打磨后正文},
    format="markdown"
  ) → 返回链接
覆盖原文 → ⚠️ 需确认后 yuque_update_doc
```

---

## 批量模式

```
触发：「把小记全打磨成文档」「批量打磨最近 N 条小记」

流程：
  1. yuque_list_notes → 获取小记列表
  2. 逐条 yuque_get_note 读完整内容
  3. 逐条深度打磨
  4. yuque_create_doc 写入指定知识库（标题从内容推断）
  5. 输出汇总报告

上限 20 条，超出提示分批。
```
