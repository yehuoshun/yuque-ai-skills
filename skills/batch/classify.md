# 智能分类打标（batch-classify）

遍历知识库所有文档，AI 逐篇分析主题 → 自动设计分类体系（≤3层）→ 重建目录树。支持原地整理和迁移整理两种模式。

## 触发

```
原地整理：「给《XX》的文档自动分类」「帮我整理一下《XX》的结构」「把《XX》按主题归类」「重新组织《XX》的目录」
迁移整理：「把《XX》整理到《YY》」
```

---

## 两种模式

| 模式 | 行为 | 触发 |
|------|------|------|
| **原地整理** | 在源库内重建目录 | 未指定目标库 |
| **迁移整理** | 读源库内容 → 分析分类 → 写入目标库新结构 | 指定了目标库（含「整理到」「迁移到」） |

---

## 阶段一：定位库

### 1.1 匹配源库

```
yuque_list_repos → 匹配：

  纯数字              → id 精确
  含 /（group/slug）   → namespace 精确
  纯英文/字母数字       → slug 精确
  含中文/混合          → 优先 slug 精确 → 未命中 name 模糊

唯一匹配 → 确认 | 多个匹配 → 让用户选 | 无匹配 → 告知不存在，结束
```

### 1.2 确定目标库

```
用户指定了目标库（含「整理到」「迁移到」）：
  → 同逻辑匹配
  → 找到 → 直接用
  → 未找到 → 标记「需新建」

用户未指定目标库：
  → 原地整理模式
```

---

## 阶段二：读取 & 分析

### 2.1 获取文档

```
yuque_list_docs(book_id=源库id) → 全部文档
提取：title | slug | id | updated_at

count == 0 →「《XXX》是空的」结束
count > 50 → 分批，每批 50 篇，告用户「共 {n} 篇，每批 50 篇，共 {p} 批」
```

### 2.2 逐篇分析

每批并发 3 篇：

```
每篇：
  yuque_get_doc(book_id=源库id, doc_ids: [文档id]) → body (Markdown)

  LLM 分析 ≤ 3000 字正文 → 输出：

  分类依据：核心主题 → 文章类型（教程/参考/笔记/方案）→ 所属领域
  路径命名：2-6字，不用「其他」「杂项」等模糊词
  系列文章（如「Python笔记01」「Python笔记02」）→ 归同一子类
  内容独立 → 新建分类，不强行塞入已有类

  {
    "doc_slug": "xxx",
    "title": "原标题",
    "category_path": ["一级", "二级", "三级"],
    "category_path_alt": ["一级", "二级"],
    "confidence": "high|medium"
  }
```

### 2.3 汇总合并

```
所有批次完成后，汇总全部 category_path：

1. 展平 → 统计每层频次
2. LLM 合并语义重复类名（如「容器」和「Docker」→ 统一）
3. 拆分过大的类（>20 篇）→ 细分
4. 命名不清晰 → 重命名
5. 输出最终分类树（≤3层）
```

### 2.4 未分类判定

```
仅以下情况归入「未分类」：
  • 文档正文为空
  • 正文 ≤10字 且无明确主题（「待补充」「TODO」「...」「占位」）

其余有内容文档，主题独立也单独成类。
```

---

## 阶段三：预览

```
📋 分类预览
模式：{原地整理 / 迁移整理}
源库：《{源库名}》({n} 篇) {→ 目标：《{目标库名}》({新建/已有})}

分析完成：{n} 篇 | {m} 大分类 | {k} 子类

新目录结构：

  ├─ {一级1}（{n1} 篇）
  │  ∟ {二级1}（{n} 篇）
  │  ∟ {二级2}（{n} 篇）
  │  ∟ ...
  ├─ {一级2}（{n2} 篇）
  │  ∟ ...
  ├─ ...
  └─ 未分类（{n} 篇）⚠️
      ∟「{title}」— 无内容

覆盖：{covered}/{total}（{pct}%）

操作：
  「确认」— 按新结构生成目录
  「加前缀」— 标题加分类标签：「[{分类名}] {原标题}」
  「改名 {旧}→{新}」— 重命名分类
  「排除 {分类名}」— 跳过不处理
  「取消」
```

超 50 篇时目录只展示一级 + 每级前 3 子类，其余标「...等 N 个」。

## 阶段四：执行

### 4.1 前置

```
迁移模式 + 目标库需新建 → yuque_create_repo
  → 响应不包含 namespace，手动构造：namespace = "{login}/{slug}"
  → login 从 config/yuque-config.json 的 group 字段取
  → 失败 → 中止，告知用户
```

### 4.2 重建目录

```
原地整理模式：
  1. yuque_list_toc → 备份原结构到报告
  2. yuque_remove_toc_node 逐节点清空（不删文档）
  3. 按分类树逐层创建节点 + 挂载文档：
     一级 → yuque_update_toc(action: "appendNode", action_mode: "sibling", target_uuid: "")
     二级 → yuque_update_toc(action: "appendNode", action_mode: "child", target_uuid: 父节点uuid)
     文档 → yuque_update_toc(action: "appendNode", action_mode: "child", target_uuid: 父节点uuid, doc_ids: [文档id])

迁移整理模式：
  1. 逐篇 yuque_get_doc(源库) → yuque_create_doc(目标库)
  2. yuque_update_toc 挂载到对应分类节点
```

### 4.3 可选：标题加前缀

```
用户选「加前缀」→
  前缀 = "[{最小一级分类名}]"
  去重已有前缀（避免重复加）
  yuque_update_doc(title="{前缀} {原标题}")
```

## 阶段五：报告

```
✅ 分类完成
模式：{原地整理 / 迁移整理}
{源库：《源库名》→ 目标：《目标库名》}
时间：{现在}

新结构：{m} 大分类 | {k} 子类 | ≤3 层
覆盖：{covered}/{total}（{pct}%）
未分类：{n} 篇 — {原因}

新目录：https://www.yuque.com/{namespace}

{迁移模式：「源库未动，原文仍在」}
{原地模式：「回滚：语雀版本历史可恢复原目录」}
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 源库为空 | 告知，结束 |
| 超 50 篇 | 每批 50 篇，告知总批数 |
| 单篇读取/分析失败 | 记录，标记，不阻断 |
| 全部分析失败 | 中止，告知原因 |
| API 限流(429) | 等 Retry-After，最多重试 3 次 |
| 目标库创建失败 | 中止 |
| 目录操作失败 | 保留已建节点，输出半成品 + 恢复指引 |
| 标题更新失败 | 记录失败，不阻断目录重建 |
