# 批量格式标准化（batch-format）

按指定格式风格批量标准化知识库文档。支持预设风格、参考文档提取、自定义提示词三种格式来源。

## 触发

```
「统一《XX》的文档格式」「批量格式化《XX》」
「把《XX》按技术文档风格整理」「按《YY》的格式统一《XX》」
「用这个风格格式化《XX》：标题用H2，代码块加语言」
「根据这篇文档的格式统一《XX》」
```

---

## 三种格式来源

| 来源 | 触发 | 怎么用 |
|------|------|--------|
| **预设风格** | 「按技术文档风格」「学术论文格式」「简洁笔记」 | 选预设模板 → 预览 → 执行 |
| **参考文档** | 「按《XX文档》的格式」「跟这篇一样排版」 | 读参考文档 → LLM 提取格式特征 → 预览 → 执行 |
| **自定义提示词** | 「按这个要求格式化：标题H2...」 | 用户描述规则 → LLM 解析 → 预览 → 执行 |

## 两种模式

| 模式 | 行为 | 触发 |
|------|------|------|
| **原地修正** | 读源库 → 格式化 → 覆盖 | 未指定目标库 |
| **复制修正** | 读源库 → 格式化 → 写入目标库（源库不动） | 指定了目标库（「放到」「写入」） |

## 预设风格

| 风格 | 适用场景 | 规则集 |
|------|---------|--------|
| **技术文档** | API 文档、技术教程、README | H2 起始、代码块语言标注、中英文空格、段落间距统一 |
| **学术论文** | 论文笔记、文献综述 | 严格标题层级、引用格式、段落缩进 |
| **简洁笔记** | 个人笔记、碎片记录 | 列表一致性、段落间距、去多余空行 |
| **项目文档** | 需求文档、设计文档 | 标题层级、表格对齐、链接规范 |

用户可自定义组合：「技术文档风格 + 去掉图片检查」。

---

## 阶段一：定位库

```
yuque_list_repos → 匹配：

  纯数字              → id 精确
  含 /（group/slug）   → namespace 精确
  纯英文/字母数字       → slug 精确
  含中文/混合          → 优先 slug 精确 → 未命中 name 模糊

唯一匹配 → 确认 | 多个匹配 → 让用户选 | 无匹配 → 告知不存在，结束
```

目标库指定时（「放到」「写入」）→ 同逻辑匹配，未找到标记「需新建」。

---

## 阶段二：解析格式来源

### A. 预设风格

```
触发词匹配 → 加载预设规则集 → 直接进阶段三

用户可增减：「技术文档风格+去掉图片检查」
```

### B. 参考文档

```
1. 解析参考文档标识（名称/id/slug/namespace/链接）
2. yuque_get_doc → 读参考文档 body
3. LLM 提取格式特征：

   分析维度：
   - 标题：起始层级、是否跳跃、编号风格
   - 代码块：语言标注、fences vs 缩进
   - 图片：alt 文本、对齐
   - 中英文：空格、标点风格
   - 列表：符号偏好（- / * / 1.）、缩进层级
   - 段落：间距、首行缩进
   - 链接：文字链接 vs 裸 URL
   - 强调：加粗范围、斜体
   - 表格：对齐、边界

4. 输出规则列表 → 进阶段三
```

### C. 自定义提示词

```
用户自然语言描述规则，如：
  「标题用 H2 开始，代码块都要有语言标注，
    中文和英文数字之间加空格，段落间只留一个空行」

LLM 解析为规则列表 → 进阶段三
```

---

## 阶段三：检测 + 预览

### 3.1 检测

```
yuque_list_docs(book_id=源库id) → 全部文档
提取：title | slug | id | updated_at
count == 0 → 告知，结束
count > 50 → 分批 50 篇/批

逐篇 yuque_get_doc → body
按解析出的规则逐项检测（纯正则快扫）
统计各规则命中篇数
```

### 3.2 预览

```
📋 格式标准化预览
源库：《{源库名}》（{n} 篇）
模式：{原地修正 / 复制修正 → 目标：《{目标库名}》}
格式来源：{技术文档风格 / 参考《{参考文档}》/ 自定义规则}

提取的规则：
  • 标题从 H2 开始，不跳级
  • 代码块加语言标注
  • 中英文/数字间加空格
  • 段落间统一单空行

影响范围：

  规则                    命中
  标题层级不一致          {n} 篇
  代码块缺语言标注        {n} 篇
  中英文缺空格            {n} 篇
  段落间距异常            {n} 篇

共 {m} 篇存在至少 1 个问题

示例 diff（前 3 篇）：

  【{title}】
  - ### 基础语法
  + ## 基础语法
  - ```
  + ```python
  - 安装Python后配置pip
  + 安装 Python 后配置 pip

操作：
  「确认」— 批量修正
  「排除 N号」— 跳过指定文档
  「加规则 {描述}」— 补充规则
  「减规则 {规则名}」— 移除规则
  「取消」
```

仅展示前 3 篇 diff 示例。

## 阶段四：执行

### 4.0 前置

```
复制修正模式 + 目标库需新建 → yuque_create_repo
  → 响应不包含 namespace，手动构造：namespace = "{login}/{slug}"
  → login 从 config/yuque-config.json 的 group 字段取
  → 失败 → 中止
```

### 4.1 逐篇修正

并发 3 篇：

```
每篇修正顺序（结构先行，内容后行）：

  1. 标题层级修正 → 正则替换
  2. 段落间距修正 → 正则替换
  3. 列表一致性修正 → 正则替换
  4. 中英文排版修正 → 正则替换
  5. 代码块语言标注（需 LLM）
  6. 图片描述补充（需 LLM）
  7. 链接规范修正（需 LLM）

  规则 5/6/7 合并为一次 LLM 调用

  原地修正：
    yuque_get_doc → 修正 → yuque_update_doc(body=新body)

  复制修正：
    yuque_get_doc(源库) → 修正 → yuque_create_doc(目标库)
```

### 4.2 目录

```
复制修正模式 → yuque_update_toc(目标库) 挂上新文档
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 源库为空 | 告知，结束 |
| 参考文档不存在 | 告知，让用户换参考 |
| 参考文档内容为空 | 告知「参考文档无内容，无法提取格式」 |
| 全库无格式问题 | 告知「格式已经很规范 🎉」 |
| 超 50 篇 | 分批检测 + 分批修正 |
| 单篇读取/更新失败 | 记录，继续 |
| API 限流(429) | 等 Retry-After，最多 3 次 |
| 目标库创建失败 | 中止 |
