# 文档合并（merge）

多篇相关文档合并为单篇完整长文，自动拼接内容、生成导航目录。

## 触发词

```
「合并」「合成一篇」「拼在一起」
「把这几篇合并」「Docker入门(一)到(五)合成一篇」
```

---

## 合并策略

### 按顺序拼接

```
默认：用户指定列表顺序 → 按序拼接

自动排序（用户未指定顺序时）：
  检测标题中的序号模式：
    「第一章」「1.」「(一)」「01」「Part 1」
    → 按序号升序排列
  无序号 → 腌制原顺序（list_docs 返回顺序）

中间分节：每篇原文 body 之间加 `---` 分隔
```

### 内容处理

```
1. 批量 yuque_get_doc → 读每篇 body
2. 第一篇完整保留
3. 后续文档：
   - 移除重复的 H1 标题（如每篇都以「Docker教程」开头）
   - 保留 H2 及以下内容
   - 可选：H2 前不加 H1，改用更强的视觉分隔
4. 拼接 + 生成总导航 TOC
5. yuque_create_doc 创建合并文档
6. 可选删除原始分篇
```

### 导航 TOC

```
合并后文档开头自动生成导航：

# {总标题}

## 📑 目录
- [第一章 xxx](#第一章)
- [第二章 xxx](#第二章)
- ...

---
{第一篇内容}
---
{第二篇内容}
...
```

## 操作流程

```
1. 定位要合并的文档：
   - 用户给定文档名列表：「合并《A》《B》《C》」
   - 搜索匹配：「合并『Docker入门』系列」
     → yuque_search 搜 title 匹配 → 列出匹配项
   - 整个目录：「合并『Java教程』目录下全部」→ 按 TOC 顺序

2. 预览合并方案：

📋 合并预览

总标题：{用户指定 / 自动推断}

合并 {N} 篇：

  #  标题            字数
  1  Docker入门(一)   1200
  2  Docker入门(二)   1860
  3  Docker入门(三)   772
  ...

合并后：约 {sum} 字

操作：
  「确认合并」
  「改标题 {新标题}」
  「排除 序号N」
  「调序」
  「不加目录」
  「取消」

3. 用户确认 → 拼接 + create_doc

4. 输出报告
```

---

## 标题推断

```
总标题生成策略：

  用户指定 → 直接用
  文档名称有公共前缀 → 取公共部分
    例：「Docker入门(一)」「Docker入门(二)」→ 总标题「Docker入门」
  无公共前缀 → 取第一篇的 H1
  都不行 → 「合并文档 {日期}」

去重处理：
  所有子文档都有的相同 H1 → 总标题使用它，子内容中去掉重复 H1
  子文档 H1 各不同 → 保留各自的 H1，总标题另行生成
```

## 输出报告

```
✅ 合并完成

合并后：《{总标题}》（{N} 字）
来源：{N} 篇文档

📝 已创建：{链接}

{如已删原始分篇：}
🗑️ 已删除原始 {N} 篇分篇文档（不可恢复，请注意）
```

## 上限保护

| 项 | 上限 |
|----|------|
| 合并文档数 | 30 篇 |
| 合并后总字数 | 无硬性上限，但超 5 万字提醒 |

---

## 错误处理

| 场景 | 处理 |
|------|------|
| 文档未找到 | 「{名称} 未找到」 |
| 只有 1 篇 | 「至少需要 2 篇文档才能合并」 |
| 合并后超 5 万字 | 「合并后约 {n} 万字，可能过长。是否继续？」 |
| 搜索匹配到 0 篇 | 「未找到匹配的文档」 |
| 创建失败 | 重试 1 次，仍失败告知用户 |
| 删除确认 | ⚠️ 二次确认「原始 {N} 篇将被硬删除，确认？」 |

## 跨 Skill 联动

```
合并前 → batch/summarize 生成每篇摘要（了解内容）
合并后 → write/polish 打磨合并后的长文
       → batch/split 是反向操作，可互补
```
