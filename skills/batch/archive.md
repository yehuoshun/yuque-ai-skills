# 批量归档（batch-archive）

把知识库里长期未更新的文档，一键搬到归档库。支持归档（移动）和备份（复制）两种模式。

## 触发词

```
归档模式：「归档」「清理」「移走」「搬」
备份模式：「备份」「复制一份」「镜像」
仅预览：「看看有哪些该归档了」「检查一下」
```

## 参数解析

从用户输入提取：

| 参数 | 必填 | 默认值 | 支持 |
|------|------|--------|------|
| 源库 | ✅ | — | 名称 / book_id / slug / namespace |
| 目标库 | ❌ | 自动「归档-{源库名}」 | 同上 |
| 过期阈值 | ❌ | 180 天 | 「N天」「N个月」「半年」「2024年之前」 |

模式判定：
- 含「归档/清理/移走/搬」→ 归档模式（移动：复制后删源库）
- 含「备份/复制一份/镜像」→ 备份模式（仅复制，源库不动）
- 含「看看/检查」且无归档/备份意图 → 仅预览，不执行

阈值解析：
- 「N天/N个月/N年」→ 换算天数（1月=30天，1年=365天）
- 「半年」→ 180天，「一年」→ 365天
- 「YYYY年之前/以前」→ 绝对日期，updated_at < 该日期
- 「YYYY年MM月DD日之前」→ 同上
- 未指定 → 默认 180 天
- 看不懂 → 告用户「未识别阈值，将使用默认 180 天」

---

## 阶段一：定位库

### 1.1 获取知识库列表

```
yuque_list_repos → 获取所有知识库
提取每个库：id | name | slug | namespace
```

### 1.2 匹配源库

```
输入格式判定 → 匹配：

  纯数字                  → id == 输入（精确）
  含 /（如 group/slug）    → namespace == 输入（精确）
  纯英文/字母数字/不含中文   → slug == 输入（精确）
  其他（含中文/混合）       → 优先 slug == 输入（精确命中）
                             未命中 → name 包含 输入（模糊）
```

匹配结果：

```
  唯一匹配 → 确认 ✅
  多个匹配 → 列出匹配项让用户选
  无匹配   → 「未找到《XXX》知识库，请确认名称后重试」结束
```

### 1.3 确定目标库

```
用户指定了目标库：
  → 同上匹配逻辑
  → 找到 → 直接用（追加模式）
  → 未找到 → 标记「需新建」

用户未指定目标库：
  → yuque_list_repos 搜 name 包含「归档-{源库name}」
  → 找到 → 直接用（追加模式）
  → 未找到 → 标记「需新建《归档-{源库name}》」
```

---

## 阶段二：扫描筛选

### 2.1 获取文档列表

```
yuque_list_docs(book_id=源库id) → 全部文档
提取：title | slug | id | updated_at
```

源库为空 →「《XXX》是空的，没有文档可归档」结束。

### 2.2 按阈值筛选

```
now = 当前时间
对每篇文档：
  相对阈值 → expired = (now - updated_at) > 阈值天数
  绝对阈值 → expired = updated_at < 截止日期

筛选出 expired == true 的文档
按过期天数降序排列
```

### 2.3 截断

```
count > 100 → 取前 100 篇，提示「共命中 {count} 篇，首批处理 100 篇，剩余可分批」
count == 0 → 「所有文档都在 {阈值} 内活跃更新中 🎉」结束
```

---

## 阶段三：预览

输出预览表，等待用户确认。

```
📋 {归档/备份}预览
源库：{源库name}（共 {total} 篇）→ 目标库：{目标库name}（{新建/已有 N 篇，本次追加}）
模式：{归档（移动）⚠️ 原库文档将被删除 / 备份（复制）源库不动}
条件：超过 {阈值描述}（截止 {日期}）
命中：{count} 篇

 #  文档              最后更新       过期天数
 1  {title}           {updated}      {n}天
 2  {title}           {updated}      {n}天
...
{count}  {title}           {updated}      {n}天

操作：
  「确认」— 开始执行
  「排除 {N}号,{M}号」— 跳过指定文档
  「改阈值 {N天/描述}」— 重新筛选
  「改目标 {名称}」— 换目标库
  「切换{备份/归档}」— 切换模式
  「取消」— 结束
```

文档数 ≤ 20 显示全部；> 20 显示前 10 和后 5，中间省略。

### 用户回应处理

| 用户操作 | 处理 |
|----------|------|
| 「确认」 | 进执行阶段。归档模式标注⚠️提示将删除 |
| 「排除 N号,M号」 | 移除指定文档，重新预览 |
| 「改阈值 N天」 | 重新筛选 → 重新预览 |
| 「改目标 名称」| 重新匹配目标库 → 重新预览 |
| 「切换备份」| 切换为备份模式 → 重新预览（去掉删除警告） |
| 「切换归档」| 切换为归档模式 → 重新预览（加上删除警告） |
| 「取消」/「算了」| 结束 |

## 阶段四：执行

### 4.0 前置

```
目标库需新建 → yuque_create_repo(name="归档-{源库name}", slug, description)
  → 响应不包含 namespace，手动构造：namespace = "{login}/{slug}"
  → login 从 config/yuque-config.json 的 group 字段取
  → 失败 → 中止，告知用户「创建目标库失败」
```

### 4.1 复制阶段

逐篇并发执行，并发数 3：

```
对每篇文档（并发 3 个）：

  Step A: yuque_get_doc(book_id=源库id, doc_id=文档id)
    → 成功：拿 body（Markdown 原文）
    → 403/404：记录失败「{title} — 读取失败 ({code})」，skip
    → 429：等 Retry-After 秒后重试，最多 3 次
    → 超时：重试 1 次，仍失败记录失败

  Step B: yuque_create_doc(
            book_id=目标库id,
            title=原标题,
            body=body,
            format="markdown"
          )
    → 成功：记录成功
    → 失败：记录失败「{title} — 写入失败 ({code})」
```

### 4.2 验证

```
yuque_list_docs(book_id=目标库id) → 确认新文档已在目标库

成功数 != 期望数 → 告用户「写入验证异常：期望 {expect} 篇，确认 {actual} 篇」
```

### 4.3 删除阶段（仅归档模式）

备份模式跳过此阶段直接进 4.4。

```
⚠️ 二次确认：
「复制验证通过，{success} 篇已全部写入《{目标库name}》。
  即将从《{源库name}》删除这 {success} 篇文档（硬删除，不可恢复）。
  最后确认？」

用户确认 →
  逐篇 yuque_delete_doc(book_id=源库id, doc_id=文档id)
    成功 → 记录成功
    失败 → 记录失败「{title} — 删除失败 ({code})」

用户取消 → 保留已复制的文档在目标库，告知用户「只完成了复制，源库未删除」
```

### 4.4 目录整理

```
目标库 → yuque_update_toc(action: "appendNode", action_mode: "sibling", ...) 挂上新文档

源库（仅归档模式，删了文档的情况）→
  遍历被删除文档对应的目录节点
  yuque_remove_toc_node → 清理源库空节点
```

## 错误处理速查

| 场景 | 处理 |
|------|------|
| 源库不存在 | 「未找到《XXX》知识库，请确认名称后重试」结束 |
| 源库为空 | 「《XXX》是空的，没有文档」结束 |
| 无过期文档 | 「所有文档都在活跃维护中 🎉」结束 |
| 目标库创建失败 | 中止，告用户检查权限 |
| 单篇读取失败(403) | 记录失败，继续 |
| 单篇读取失败(404) | 记录失败「文档可能已被删除」，继续 |
| 单篇写入失败 | 记录失败，继续 |
| API 限流(429) | 等 Retry-After 秒，最多重试 3 次 |
| 网络超时 | 重试 1 次，仍失败记录 |
| 批量超过 100 篇 | 只处理前 100，提示剩余数量 |
| 用户中途取消 | 已完成的保留，未完成的不动 |
