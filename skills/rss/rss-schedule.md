# yuque_rss_schedule

> 端点：无（RSS 定时抓取策略分析）
> 说明：主方案读取 RSS 配置知识库，分析最近更新频率，生成推荐下次抓取时间并写回。兜底方案使用 KV 去重数据。

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `source` | string | ✅ | 数据源 key，如 `cnblogs` |
| `kv_namespace` | string | ❌ | KV namespace（兜底方案用），默认等于 source |
| `mode` | string | ❌ | `analyze`（分析+写回，默认）/ `dry_run`（仅分析不写） |
| `raw` | boolean | ❌ | 返回原始 JSON（默认 false） |

## 频率分档（保守策略）

| 分档 | 条件 | 建议间隔 |
|------|------|----------|
| 高频 | 近 7 天 ≥5 篇 | 每天 1 次 |
| 中频 | 近 7 天 1-4 篇 | 每 7 天 |
| 低频 | 近 14 天 1 篇 | 每 15 天 |
| 休眠 | 近 30 天 0 篇 | 每 30 天 |

## 主方案：配置知识库

在 `config.json` 中配置 `rss.schedule.book_id` 指定 RSS 配置知识库：

```json
{
  "rss": {
    "enabled": true,
    "schedule": {
      "book_id": 80278170
    }
  }
}
```

工具调用后自动：
1. 读取 KV 去重数据，统计最近更新频率
2. 按分档规则计算推荐抓取间隔
3. 写入/更新配置知识库中的文档（每 feed 一篇）

文档格式（Markdown）：

```markdown
# 博客园/首页最新

- 数据源: cnblogs
- Feed: sitehome
- 上次抓取: 2026-06-18 12:00
- 推荐下次抓取: 2026-06-19
- 抓取间隔: 1天
- 频率分档: 高频
- 近7天文章数: 15
- 近14天文章数: 28
- 近30天文章数: 45
- 活跃作者: Johny_Zhao, Newbe36524, ...
```

## 兜底方案：KV 去重数据

未配置 `schedule.book_id` 时自动回退到纯 KV 分析，仅输出分析结果不写回。

## 使用场景

- 分析 RSS 源的最近更新频率，规划定时抓取间隔
- 在设置 cron 定时任务前，先了解最佳抓取频率
- 定期运行（如每天一次），动态更新推荐抓取时间
- 人工查看配置知识库，快速了解各 feed 状态

## 前置条件

- KV 功能已启用
- KV namespace 中有已抓取的文章记录（v2.6.0+ 格式，含时间戳和作者）
- 主方案需额外配置 `rss.schedule.book_id`

## 注意事项

- 以最近更新情况为准，不看历史总平均
- 数据量越大分析越准确，建议积累至少 10 篇以上
- 旧格式 KV 数据（v2.5.0 及之前）不含时间戳，会提示重新抓取
- 最小抓取间隔为 1 天
