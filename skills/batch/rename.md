# 批量重命名（batch-rename）

按指定规则批量修改文档标题。支持前缀/后缀/序号/查找替换/正则/模板/移除等七种方式。

## 触发

```
「给《XX》的文档批量改名」「统一《XX》的文档命名」
「把《XX》所有文档前面加序号」「给《XX》标题加日期前缀」
「把《XX》里的『xxx』改成『yyy』」「按命名规范重命名《XX》」
「把《XX》改名后放到《YY》」
```

---

## 支持方式

| 方式 | 示例输入 | 效果 |
|------|---------|------|
| **加前缀** | 「加日期前缀」「前缀加 [已归档]」 | `文档` → `2026-05-23 文档` |
| **加后缀** | 「加版本号 v2」 | `文档` → `文档 v2` |
| **序号前缀** | 「加序号」「前面加 01-」 | `文档A` → `01-文档A` |
| **查找替换** | 「把『旧名』改成『新名』」 | `旧名相关` → `新名相关` |
| **正则替换** | 「去掉日期 `\\d{4}-\\d{2}-\\d{2}`」 | `2024-01 笔记` → `笔记` |
| **模板** | 「格式：{date}-{category}-{title}」 | 变量替换 |
| **移除内容** | 「去掉『已废弃』前缀」「删除括号内容」 | `[旧] 笔记 (废弃)` → `笔记` |

## 两种模式

| 模式 | 行为 | 触发 |
|------|------|------|
| **原地重命名** | 在源库内改标题 | 未指定目标库 |
| **复制重命名** | 读源库 → 新标题写入目标库（源库不动） | 指定了目标库 |

---

## 阶段一：定位库

```
yuque_list_repos → 匹配：

  纯数字              → id 精确
  含 /（group/slug）   → namespace 精确
  纯英文/字母数字       → slug 精确
  含中文/混合          → 优先 slug 精确 → 未命中 name 模糊

唯一匹配 → 确认 | 多个匹配 → 让用户选 | 无匹配 → 告知不存在，结束
```

目标库指定时同逻辑匹配。

---

## 阶段二：扫描 & 生成新标题

### 2.1 获取文档

```
yuque_list_docs(book_id=源库id) → 全部文档
提取：title | slug | id | updated_at | created_at

count == 0 → 告知，结束
count > 50 → 分批 50 篇/批
```

### 2.2 生成新标题

根据用户指定的方式逐篇生成：

```
加前缀 → 前缀 + 原标题
加后缀 → 原标题 + 后缀
序号前缀 → {序号} - 原标题
  排序规则：
    用户指定 → 按用户说的排
    未指定 + 有 created_at → 按创建时间升序
    未指定 + 无时间 → 按原标题字母序
查找替换 → title.replace(旧, 新)
正则替换 → title.replace(regex, 新)
模板 → 解析变量组合
  支持变量：
    {date}     = created_at 日期（YYYY-MM-DD）
    {time}     = created_at 时间（HH:mm）
    {title}    = 原标题
    {index}    = 排序序号（从 1 开始，补零到最大位数）
移除内容 → 去掉匹配的前缀/后缀/括号内文字
```

### 2.3 同名检测

```
生成新标题后去重：
  无冲突 → 通过
  有冲突 → 追加 (1) (2) 后缀 → 预览中醒目标注 ⚠️
```

---

## 阶段三：预览

```
📋 重命名预览
源库：《{源库名}》（{n} 篇）
模式：{原地重命名 / 复制重命名 → 目标：《{目标库名}》}
方式：{序号前缀 + 日期前缀}

排序：按创建时间升序

 #  原标题                      新标题
 1  Docker部署笔记              01-2024-03-15 Docker部署笔记
 2  React入门                   02-2024-05-20 React入门
 3  Python爬虫                  03-2024-06-01 Python爬虫
 4  笔记                        04-2024-06-01 笔记 ⚠️ 同名已追加序号

同名冲突：1 篇（第 4 篇，已自动追加序号）

操作：
  「确认」— 批量重命名
  「排除 N号」— 跳过指定文档
  「调序 3号→1号」— 调整排序
  「改前缀 {新前缀}」— 调整规则
  「取消」
```

超 20 篇仅展示前 20，标「...等共 N 篇」。

---

## 阶段四：执行

### 4.0 前置

```
复制重命名模式 + 目标库需新建 → yuque_create_repo
  → 响应不包含 namespace，手动构造：namespace = "{login}/{slug}"
  → login 从 config/yuque-config.json 的 group 字段取
  → 失败 → 中止
```

### 4.1 逐篇执行

并发 3 篇：

```
原地重命名：
  yuque_update_doc(doc_id=文档id, title=新标题)

复制重命名：
  yuque_get_doc(源库) → body
  yuque_create_doc(目标库, title=新标题, body=body)

同名冲突：
  目标库中已存在同名 → 追加 (N) 后缀直到唯一
  如「文档」→「文档 (1)」→「文档 (2)」
```

### 4.2 目录

```
复制重命名模式 → yuque_update_toc(目标库) 挂载新文档
原地重命名模式 → 标题变但目录索引自动跟随，无需手动操作
```

```
{复制：「源库未动，目标库为改名副本」}
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 源库为空 | 告知，结束 |
| 重命名规则无法解析 | 告知「未理解命名规则，请换个说法或给出示例」 |
| 超 50 篇 | 分批预览 + 分批执行 |
| 单篇更新失败 | 记录，继续 |
| 目标库创建失败 | 中止 |
| API 限流(429) | 等 Retry-After，最多 3 次 |
