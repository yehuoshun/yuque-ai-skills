# 批量整理（reorganize）

把知识库 A 的文档批量复制到知识库 B，**源库不动**（只复制，不删除）。

> ⚠️ **关键区别**：
> - **整理**（本 skill）→ 复制到目标库，源库不动
> - **归档/迁移**（archive.md）→ 复制后删除源库文档

## 触发词

```
「整理到」「搬到」「复制到」「A库整理到B库」「分到」「归类到」
```

不触发：「归档」「迁移」「清理」「移走」→ 走 archive.md

## 参数解析

从用户输入提取：

| 参数 | 必填 | 默认值 | 支持 |
|------|------|--------|------|
| 源库 | ✅ | — | 链接 / 名称 / book_id / slug / namespace |
| 目标库 | ✅ | — | 同上 |

模式判定：
- 含「整理到/搬到/复制到/归类到/分到」→ 整理模式（复制，源库不动）
- 含「归档/清理/移走」→ 不触发本 skill，走 archive.md

标识符解析（链接优先）：
- **语雀链接**（如 `https://www.yuque.com/yehuoshun/fzmebf`）→ 提取 namespace `yehuoshun/fzmebf`
- **namespace**（含 `/`，如 `yehuoshun/fzmebf`）→ 直接匹配
- **纯数字** → id == 输入（精确）
- **纯英文/字母数字/不含中文** → slug == 输入（精确）
- **含中文/混合** → 优先 slug 精确 → 再 name 模糊

## 阶段一：定位库

### 1.1 获取知识库列表

```
yuque_list_repos → 获取所有知识库
提取每个库：id | name | slug | namespace
```

### 1.2 匹配源库

```
输入格式判定 → 匹配：

  语雀链接                → 提取 namespace，精确匹配
  含 /（如 group/slug）    → namespace == 输入（精确）
  纯数字                  → id == 输入（精确）
  纯英文/字母数字/不含中文   → slug == 输入（精确）
  其他（含中文/混合）       → 优先 slug == 输入（精确命中）
                             未命中 → name 包含 输入（模糊）
```

匹配结果：

```
  唯一匹配 → 确认 ✅
  多个匹配 → 列出匹配项让用户选
  无匹配   → 「未找到《XXX》知识库，请确认名称后重试」结束
```

### 1.3 匹配目标库

同 1.2 匹配逻辑。目标库无默认值，必须明确指定。

### 阶段二：预览

```
📋 整理预览
源库：{源库name}（共 {total} 篇）→ 目标库：{目标库name}
模式：📋 复制（源库不动）
命中：{count} 篇
```

用户确认后执行。

### 阶段三：执行

**方案 A：一步到位（推荐）**

```
MCP 工具：yuque_batch_mount_toc(
  book_id = 目标库ID,
  categories = {分类名: [doc_id, ...]},
  parent_uuid = 可选父节点UUID,
  batch_size = 100
)
```

自动完成：创建 TITLE 节点 → 批量挂载文档到分类下。

**方案 B：目录已建好，只挂文档**

```
MCP 工具：yuque_batch_mount_to_existing_toc(
  book_id = 目标库ID,
  mapping = {分类名: {uuid: TITLE_UUID, doc_ids: [doc_id, ...]}}
)
```

**方案 C：手动控制（旧方式）**

如果源库有目录结构需要复制到目标库，分两步：
1. `yuque_get_toc_flat` 读取源库目录结构
2. `yuque_update_toc` 在目标库重建目录
3. `yuque_copy_docs_cross_book` 复制文档（按分类分组传 doc_ids）

### 阶段四：报告

```
✅ 整理完成
源库：{源库name} → 目标库：{目标库name}
模式：📋 复制（源库未动）
成功：{N} 篇 | 失败：{K} 篇
目标库：https://www.yuque.com/{namespace}
```

## 铁律

🚫 **整理操作不删除源库任何文档。** 如需要删除源库，使用 archive.md（归档/迁移）。

## 错误处理

| 场景 | 处理 |
|------|------|
| 源库不存在 | 「未找到源库」结束 |
| 源库为空 | 「源库没有文档」结束 |
| API 限流(429) | 内部重试，持续失败告用户 |
| 单篇写入失败 | 记录失败，继续下一篇 |
