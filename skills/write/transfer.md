# 风格迁移（transfer）

用参考文档的风格重写目标文档，保留事实内容，只换表达方式。

## 触发词

```
「用{xx}的风格改写」「模仿{xx}的文风」「参照{xx}的写法」
「把这段笔记写成{某文档}那样的风格」
```

---

## 输入

```
目标文档（被改写的内容）+ 风格参考文档（模仿其风格）

指定方式：
  「用《{风格参考}》的风格改写《{目标文档}》」
  「把这段笔记写成{某文档}那样的风格」
  「模仿这个知识库的风格」
```

---

## 流程

```
1. yuque_get_doc 读风格参考文档 → 提取风格画像（10 维分析，见 analyze.md）
2. yuque_get_doc 读目标文档 → 提取核心事实和观点
3. LLM 用风格画像的规则重写目标文档的核心事实
4. 输出改写结果 + 风格对照说明
```

---

## 约束

```
- 不改变目标文档的事实信息和核心观点
- 不添加目标文档没有的内容
- 风格参考只影响表达方式，不影响内容
- 如风格参考和目标文档类型差异过大（如诗歌→技术文档），提示并降级为轻度风格化
```

---

## 输出格式

```
## 🎭 风格迁移

**目标文档**：《{A}》
**风格参考**：《{B}》
**风格匹配度**：{高/中/低}

---
{改写后正文}

---
📊 风格对照：
| 维度 | 原文 | 迁移后 |
|------|------|--------|
| 语气 | ... | ... |
| 句式 | ... | ... |
```
