# 批量智能摘要（batch-summary）

对单篇/多篇/整个知识库自动生成 4 级粒度摘要。可选输出为新文档或更新原文。

## 触发词

```
单篇：「总结一下」「这篇讲了什么」「概括一下」「摘要」
多篇：「把这几篇总结一下」「搜XX然后总结」「总结最近N篇」
知识库：「给知识库做个概览」「每篇写个简介」「知识库目录摘要」
指定粒度：「一句话总结」「详细摘要」「按技术方案模板总结」
```

## 参数解析

从用户输入提取：

| 参数 | 必填 | 默认值 | 支持 |
|------|------|--------|------|
| 目标 | ✅ | — | 文档名/slug/doc_id / 知识库名 / 搜索词 |
| 粒度 | ❌ | 自动推荐 | `L1`/`L2`/`L3`/`L4` / 「一句话」「段落」「详细」「结构化」 |
| 输出方式 | ❌ | 直接返回 | 「直接返回」「存成文档」「更新原文」 |
| 关注维度 | ❌ | 全维度 | 技术方案/业务逻辑/学习要点/行动建议 等 |
| 目标库 | ❌ | 默认库 | 存成文档时的目标知识库（名称/book_id） |
| 上限 | ❌ | 50 | 多篇/知识库模式最大处理篇数 |

### 粒度解析

```
「一句话」「L1」「简短」     → L1（≤50 字）
「段落」「L2」「简介」       → L2（100-200 字）
「详细」「L3」「深度」       → L3（300-800 字）
「结构化」「L4」「模板」     → L4（按文档类型匹配模板）
未指定 /「自动」            → 自动推荐（LLM 看长度和内容类型判断）
```

### 模式判定

```
含文档名/slug/doc_id + 不含「都/全部/所有/知识库」 → 单篇模式
含「搜索/搜」 + 「总结」 → 多篇模式（先搜索再摘要）
含「这几篇/这些」 + 文档列表 → 多篇模式
含「知识库/全部/所有/每篇/概览/目录摘要」 → 知识库模式
```

### 输出方式解析

```
含「存成文档」「写入」「创建文档」 → 创建新文档
含「更新原文」「写回」「追加」     → 更新原文
未指定 /「直接」                   → 直接返回
```

### 关注维度解析

```
用户指定：「重点看{维度1}、{维度2}」 → 提取到维度列表
未指定 → 全维度（核心论点、关键证据、技术要点、行动建议）

常见维度：
  技术文档：架构设计、实现方案、性能指标、注意事项
  业务文档：业务流程、关键规则、角色权限、数据流转
  学习笔记：核心概念、知识点、实践技巧、常见误区
  会议纪要：决议事项、责任人、截止时间、遗留问题
  通用：核心论点、关键证据、行动建议、风险提示
```

## 四级粒度定义

### L1 一句话（≤50 字）

单句说清：这是什么、核心结论是啥。

```
Prompt 模板：
用一句话（不超过 50 字）概括以下文档的核心内容。
格式：「{文档标题}：{一句话概括}」

{body}
```

### L2 段落（100-200 字）

短段落，涵盖：主题、核心论点、关键结论。

```
Prompt 模板：
用 100-200 字概括以下文档。涵盖：主题背景、核心论点、关键结论。
格式：自然段落，不列点。

{body}
```

### L3 详细（300-800 字）

分节摘要，保留关键细节。

```
Prompt 模板：
用 300-800 字详细概括以下文档。按以下结构：
1. 概述（背景+目的）
2. 核心内容（分要点展开）
3. 关键结论/要点
4. （可选）疑问/待确认事项

{body}
```

### L4 结构化

根据文档类型匹配模板。

```
自动判定类型 → 匹配模板：

技术方案文档：
## 摘要
- **方案名称**：
- **解决问题**：
- **核心思路**：
- **技术选型**：
- **关键指标**：
- **风险与限制**：
- **实施建议**：

会议纪要：
## 摘要
- **会议主题**：
- **参与人**：
- **决议事项**：
  1. ...
- **待办 & 责任人**：
  1. @xxx — 截止 YYYY-MM-DD
- **遗留问题**：

通用结构化：
## 摘要
- **主题**：
- **核心观点**：
  1. ...
- **支撑证据**：
  1. ...
- **行动建议**：
  1. ...

Prompt 模板：
分析以下文档类型，按对应模板输出结构化摘要。只填有明确依据的字段，无依据的留空或标「无」。

{body}
```

## 阶段一：定位源

### 1.0 获取知识库列表

```
yuque_list_repos → 提取：id | name | slug | namespace
```

### 1.1 单篇模式 — 定位文档

```
用户输入是文档名/slug/doc_id →

  doc_id（纯数字） → yuque_get_doc(book_id={默认库}, doc_id={输入})
    命中 → 直接进入阶段三
    404 → 遍历所有知识库逐个试

  文档名/slug →
    1. yuque_search(query={输入}, scope={默认库}) 搜索匹配
       → 唯一匹配 → 进阶段三
       → 多个匹配 → 列出让用户选
       → 无匹配 → 扩大 yuque_search(query={输入}) 不传 scope 搜全库

    2. 仍未命中 → 「未找到《{输入}》，请确认文档名称后重试」结束
```

### 1.2 多篇模式 — 定位文档列表

```
含明确文档名列表：
  → 对每篇执行 1.1 单篇定位流程
  → 汇总去重

含搜索词（「搜 X 然后总结」）：
  → 提取搜索词
  → yuque_search(query={搜索词}, scope={可选知识库})
  → 返回前 N 篇（最多 50）
  → 展示列表：「找到 {N} 篇匹配文档，开始生成摘要？」

含「最近 N 篇」：
  → yuque_list_docs(book_id={默认库}, limit=N)
  → 按 updated_at 降序取前 N 篇

含「这几篇」无文档名：
  → 「请提供要总结的文档名称或搜索词」
```

### 1.3 知识库模式 — 定位知识库

```
知识库名称匹配（同 batch-archive 匹配逻辑）：
  纯数字 → id 精确
  含 / → namespace 精确
  纯英文不含中文 → slug 精确
  含中文 → 优先 slug，降级 name 模糊

  唯一 → 确认 ✅
  多个 → 列出
  无 → 「未找到《{name}》知识库」

确认后：
  yuque_list_docs(book_id={book_id}, limit=50)
  文档数 > 50 → 「共 {total} 篇，首批处理 50 篇，剩余可分批」
  文档数 == 0 → 「《{name}》没有文档」结束
```

## 阶段二：读取文档

```
对每篇目标文档，取 body：

  并发 3 篇：
    yuque_get_doc(book_id, doc_id) → 拿 body 字段（Markdown 原文）

  失败处理：
    403 → 记录失败，继续
    404 → 记录失败「文档可能已删除」
    429 → 等 Retry-After，最多重试 3 次
    超时 → 重试 1 次，仍失败记录
```

---

## 阶段三：生成摘要

### 3.1 单篇/多篇模式 — 逐篇生成

```
每篇调用 LLM：
  根据粒度和关注维度生成摘要

L1-L3 用对应 Prompt 模板
L4 先用 LLM 判断文档类型，再匹配模板

并发：1 篇（LLM 调用，避免超长上下文）
```

### 3.2 知识库模式 — 逐篇 + 全景概览

```
逐篇：
  每篇生成 L2 摘要（默认粒度）
  上限 50 篇

全景概览：
  汇总所有 L2 摘要 → LLM 生成知识库全景概览：

## 知识库全景概览：《{name}》
- **主题分布**：列出 Top N 主题及占比
- **覆盖范围**：已覆盖的知识域
- **知识空白**：明显缺失的领域
- **文档质量**：长文/短文分布、更新频率
- **建议**：值得优先阅读的文档推荐（Top 3-5）
```

---

## 阶段四：输出

### 4.1 直接返回（默认）

```
单篇：
## 📄 {文档标题}
{L2 摘要}
{来源链接}

多篇：
## 📋 文档摘要（{N} 篇）
{每篇的标题 + 摘要}

## 📊 综合摘要
{多篇综合摘要}

知识库：
## 📚 知识库概览：《{name}》（{N} 篇）
{全景概览}

## 📄 文档摘要列表
{每篇标题 + L2 摘要}
```

### 4.2 创建新文档

```
yuque_create_doc(
  book_id={目标库或默认库},
  title="{知识库概览/文档名} — AI 摘要",
  body={摘要内容},
  format="markdown"
)
→ 自动挂 TOC
→ 返回链接
```

### 4.3 更新原文

```
⚠️ 确认：「即将在《{title}》末尾追加 AI 摘要段落，确认？」

用户确认 →
  yuque_get_doc → 拿原 body
  yuque_update_doc(
    book_id,
    doc_id,
    body="{原body}\n\n---\n\n## 🤖 AI 摘要（生成于 {时间}）\n{摘要内容}"
  )
```

## 错误处理速查

| 场景 | 处理 |
|------|------|
| 文档未找到 | 「未找到文档，检查名称或 ID 后重试」 |
| 知识库未找到 | 「未找到知识库，检查名称后重试」 |
| 空知识库 | 「知识库为空，没有可摘要的文档」 |
| 读取失败(403) | 记录失败，继续 |
| 读取失败(404) | 记录失败「文档可能已删除」 |
| 读取失败(429) | 等 Retry-After，最多重试 3 次 |
| 超过 50 篇 | 「共 {total} 篇，首批处理 50 篇，剩余可分批」 |
| LLM 摘要失败 | 重试 1 次，仍失败用标题兜底「摘要生成失败」 |
| 创建文档失败 | 降级为直接返回 |
| 更新原文失败 | 降级为直接返回摘要内容 |
| 用户取消 | 已完成的保留 |
