# 知识库运营仪表盘（batch-dashboard）

拉团队统计 API 生成知识库运营报告。纯只读，不写操作。

## 触发

```
「生成《XX团队》的知识库周报」「知识库运营数据」
「团队知识库概览」「成员贡献排行榜」
「《XX》的文档增长趋势」「知识库健康度报告」
「帮我做知识库月度总结」
```

---

## 四个指标维度

| 维度 | 数据来源 | 展示 |
|------|---------|------|
| 整体概览 | `yuque_get_group_stats` | 总量、知识库数、成员数、今日/本周增量 |
| 知识库排行 | `yuque_get_book_stats` | 各库文档数、活跃度 |
| 成员贡献 | `yuque_get_member_stats` | 每人文档数、活跃、贡献占比 |
| 文档趋势 | `yuque_get_doc_stats` | 各库新增/更新趋势 |

---

> ⚠️ 统计 API 仅支持团队（group）账户，个人账户返回 404。Token 需勾选 `statistic:read` 权限。

---

## 场景一：自动周报

### 流程

```
1. 解析参数：团队名（必填）、时间范围（默认最近 7 天）

2. 并行拉数据（4 个 API 同时请求）：
   yuque_get_group_stats → 整体数据
   yuque_get_book_stats → 分库数据
   yuque_get_member_stats → 成员数据
   yuque_get_doc_stats → 文档趋势

3. LLM 整合：
   - 计算环比（本周 vs 上周）
   - 识别异常（某库暴涨/暴跌、某成员活跃度骤降）
   - 提取关键亮点

4. 输出报告
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 统计 API 无权限(403) | 告知「统计功能需要 statistic:read 权限，请在 Token 设置中开启」 |
| 团队无数据 | 告知「该团队暂无统计数据」 |
| 单个 API 失败 | 标注该维度不可用，其他正常输出 |
| 全部分失败 | 告知原因 |
| API 限流(429) | 等 Retry-After |
