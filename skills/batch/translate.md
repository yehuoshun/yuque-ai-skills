# AI 批量翻译（translate）

批量多语言翻译知识库文档，保留 Markdown 格式和代码块，纯 LLM 翻译。

## 触发词

```
「翻译」「翻成英文」「日文版」「多语言」
「把这个知识库翻译成英文」「这篇翻成英文和日文」
「翻译新增的文档」
```

---

## 四种模式

| 模式 | 输入 | 输出 | 场景 |
|------|------|------|------|
| 📄 单篇翻译 | 1 篇文档 + 目标语言 | 翻译后文档 | 「这篇翻成英文」 |
| 📚 批量翻译 | 知识库 + 目标语言 | 逐篇翻译存入目标库 | 「整个知识库翻成英文」 |
| 🌐 多语言 | 1 篇文档 + 多个语言 | 每语言一篇 | 「翻成英文、日文、韩文」 |
| 🔄 增量翻译 | 知识库 + 「新文档」 | 仅翻译上次后新增的 | 「翻译新增的文档」 |

---

## 语言支持

```
语雀支持的 10 种语言：

  en（英文）  ja（日文）  ko（韩文）  zh-Hant（繁体中文）
  fr（法文）  de（德文）  es（西班牙文）  pt（葡萄牙文）
  ru（俄文）  ar（阿拉伯文）

简写自动映射：
  「英文」「英语」 → en
  「日文」「日语」 → ja
  「繁体」「繁体中文」 → zh-Hant
  「韩文」「韩语」 → ko
  其他未识别 → 用户确认
```

---

## 核心规则

```
1. 保留所有 Markdown 格式标记（# -* ``` > |表格| [链接]()）
2. 代码块内容不翻译（代码是通用的）
3. URL 链接不翻译
4. 表格只翻译文字内容，保持列结构
5. 专有名词默认不翻译（可配置）
6. 技术文档用正式语体，个人笔记可口语化
```

### 专有名词策略

```
默认：不翻译专有名词（技术文档更安全）
  → SpringBoot、Docker、Kubernetes、API、HTTP 等保持原文

用户激活：
  「专有名词也翻译」「全翻」 → LLM 自行判断翻译
  「SpringBoot 不翻，其他都翻」 → 指定保留名单
```

---

## Prompt 模板

```
将以下 Markdown 文档翻译为{目标语言}。

规则：
  1. 保留所有 Markdown 格式标记（#、-、>、```、表格 | 等）
  2. 代码块内容不翻译
  3. URL 链接不翻译
  4. {专有名词规则}
  5. 风格：{风格描述}

原文：
{body}

翻译：
```

---

## 输出方式

```
默认：yuque_create_doc 存入源库

标题规则：
  {原标题} ({语言代码})
  例：Docker入门 (en)

多语言：每语言独立创建
  例：Docker入门 (en) / Docker入门 (ja)

目标库：
  用户指定 → 存指定库
  未指定 → 存源文档所在库
  「新建翻译库」 → yuque_create_repo("翻译-{源库名}") → 存新库

slug：{原slug}-{lang}，避免冲突
```

---

## 模式一：📄 单篇翻译

```
1. 定位目标文档（同其他 skill 的文档定位逻辑）
2. yuque_get_doc 读 body
3. LLM 翻译
4. yuque_create_doc 存翻译版
5. 返回链接
```

---

## 模式二：📚 批量翻译

```
触发：「把{知识库}翻译成{语言}」

1. yuque_list_repos → 定位源库
2. yuque_list_docs → 获取全部文档（上限 30 篇/批）

3. 并发处理（3 并发）：
   每篇：yuque_get_doc 读 body → LLM 翻译 → yuque_create_doc 写入

4. 输出汇总报告
```

### 汇总报告

```
✅ 翻译完成 — {源库名} → {目标语言}

📊 {N} 篇 | 成功 {K} | 失败 {F} | 跳过 {S}

✅ 成果：
  1. {标题} ({lang}) → {链接}
  2. ...

❌ 失败（如有）：
  《{title}》 — {原因}
```

---

## 模式三：🌐 多语言

```
触发：「翻成英文、日文、韩文」

1. 解析目标语言列表（逗号/顿号/空格分隔）
2. 单篇翻译模式 × N 种语言
3. 每种语言独立创建文档
4. 最多 5 种语言
```

---

## 模式四：🔄 增量翻译

```
触发：「翻译新增的文档」

1. 检查上次翻译时间戳
   → 有记录：筛选 content_updated_at > 上次时间
   → 无记录：提示「未找到上次翻译记录，是否改为全量翻译？」

2. 仅翻译变更文档

3. 更新翻译时间戳（写入图谱文档或 _meta 记录）

推荐记录方式：
  图谱文档中追加一行：
  | translate | {源库} → {lang} | {时间} | {N}篇 |
```

## 参数解析

```
用户输入提取：

  目标：
    文档名/slug/doc_id → 单篇/多语言模式
    知识库名 → 批量/增量模式

  目标语言：
    「翻译成{语言}」→ 提取语言字段
    多语言：「翻译成{A}、{B}、{C}」→ 逗号分隔

  模式：
    含「新增/增量/新的」→ 增量模式
    含多个「{语言}和{语言}」→ 多语言模式
    含知识库名 → 批量模式
    含文档名且单一语言 → 单篇模式

  专有名词：
    含「专有名词不翻译」「术语保留」→ 默认（不翻）
    含「全翻」「专业名词也翻译」→ 全部翻译

  目标库：
    含「放到{库名}」→ 指定库
    「新建翻译库」→ 创建新库
    未指定 → 源库
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 文档未找到 | 「未找到文档」 |
| 知识库为空 | 「知识库是空的」 |
| 不支持的/未识别语言 | 「支持的语言：{列表}，请选择其一」 |
| LLM 翻译后格式损坏 | 重试 1 次，在 prompt 中强调格式保留 |
| API 限流(429) | 等 Retry-After，最多重试 3 次 |
| 批量超过 30 篇 | 「共 {n} 篇，首批 30 篇，剩余可继续」 |
| 增量无记录 | 询问是否改为全量翻译 |
