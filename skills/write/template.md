# 模板写作（template）

用户提供大纲/要点，LLM 按预设模板填充为完整文档。

## 触发词

```
「按技术方案模板写」「生成周报」「写个复盘报告」
「写会议纪要」「生成 API 文档」「写 PRD」
```

---

## 预设模板

### 技术方案

```
## 技术方案：{标题}
### 背景与目标
### 方案对比
| 方案 | 优点 | 缺点 | 复杂度 |
### 推荐方案
### 实施计划
| 阶段 | 内容 | 时间 | 产出 |
### 风险与对策
| 风险 | 影响 | 概率 | 对策 |
### 排期
| 里程碑 | 日期 | 责任人 |
```

### 周报

```
## {日期} 周报 — {姓名}
### 本周产出
### 关键进展
### 问题与阻塞
### 下周计划
```

### 复盘报告

```
## 项目复盘：《{项目名}》
### 项目概述
### ✅ 做得好的（Keep）
### ⚠️ 待改进（Improve）
### ❌ 停止做（Stop）
### 🚀 行动计划
```

### 会议纪要

```
## 会议纪要：{主题}
- 时间： | 地点： | 主持人：
- 参会人：
### 议题与讨论
### 决议事项
| # | 决议 | 依据 |
### 待办事项
| # | 事项 | 责任人 | 截止日期 |
```

### API 文档

```
## {接口名称}
- **路径**：`{method} {path}`
- **描述**：
### 请求参数
| 参数 | 类型 | 必填 | 说明 |
### 请求示例
```json
```
### 返回示例
```json
```
### 错误码
| 错误码 | 说明 |
```

### PRD

```
## 产品需求文档：《{功能名}》
### 背景
### 用户故事
| 作为 | 我想要 | 以便 |
### 功能清单
| # | 功能 | 优先级 | 说明 |
### 验收标准
### 非功能需求
### 排期
```

---

## 流程

```
1. 用户提供模板类型 + 要点
   「按技术方案模板写，要做用户登录模块，JWT + Redis，支持微信登录」
2. LLM 解析要点 → 匹配模板字段 → 填充内容
3. 缺失字段标注「[待补充]」
4. 输出 → 可选创建文档
```

---

## Prompt 模板

```
根据以下要点，按{模板名称}模板生成完整文档。

要点：
{user_input}

规则：
1. 有明确依据的字段直接填充
2. 可合理推断的字段填充并标注「(推断)」
3. 完全无法确定的字段填「[待补充]」
4. 保持模板格式完整，不删除任何字段

输出：
```
