# 阅读摘录（digest）

从文章中提取核心观点、金句、行动项，生成结构化阅读笔记。支持知识卡片和批量模式。

## 触发词

```
「提取观点」「做阅读笔记」「这篇文章讲了什么」
「摘录一下」「整理成阅读笔记」「金句提取」
「行动项」「这篇文章有什么可执行的」
「生成卡片」「Zettelkasten」
```

---

## 三种模式

| 模式 | 输入 | 输出 | 触发词 |
|------|------|------|--------|
| 📖 完整摘录 | 1 篇文档 | 观点 + 证据 + 金句 + 行动项 + 疑思 | 「摘录」「阅读笔记」 |
| 🎯 快速观点 | 1 篇文档 + 速览意图 | 3-5 条核心论点 | 「只看观点」「快速」 |
| 🏷️ 知识卡片 | 1 篇文档 + 卡片意图 | Zettelkasten 卡片 | 「卡片」「Zettelkasten」 |

---

## 五维输出

| 维度 | 说明 | 产出 |
|------|------|------|
| 📌 核心观点 | 作者明确主张的独立论点 | 3-5 条，每条 30-80 字 |
| 🔍 关键证据 | 支撑论点的数据、实验、案例 | 若有，每条 20-60 字 |
| 💬 金句 | 可直接引用的原文原句 | 标注段落位置 |
| 🚀 行动项 | 读者可执行的具体动作 | 若有，每条 20-50 字 |
| 🤔 疑思 | 可质疑/进一步探索的点 | 若有，每条 20-40 字 |

### 维度生成规则

```
观点：必出。无明确观点 → 提取核心信息作为观点。
证据：有数据/实验/引用 → 出。纯观点无支撑 → 标注「无具体证据」。
金句：有精彩原文表述 → 出。全篇平淡 → 不出（不硬凑）。
行动项：教程/方法论/工具类 → 出。纯理论/纯介绍 → 不出。
疑思：有逻辑矛盾/未证伪假设 → 出。无 → 不出。
```

### Prompt 模板

```
从以下文档中提取阅读笔记。输出结构化 JSON。

规则：
1. 观点：作者明确主张的独立论点，不要混入你自己的评论
2. 证据：可验证的数据、实验、引用，无则填空数组
3. 金句：原文精彩语句，加引号，标注段落位置（如「第3段」）。无则填空数组
4. 行动项：读者看完后可以立刻做的事。纯理论/介绍类无则填空数组
5. 疑思：逻辑矛盾、未证伪假设、值得深挖的点。无则填空数组
6. 禁止杜撰

输出 JSON：
{
  "title": "原标题",
  "summary": "一句话概括（≤50字）",
  "core_ideas": ["观点1", "观点2"],
  "evidence": ["证据1"],
  "quotes": ["「原文语句」— 段落/章节"],
  "actions": ["行动1"],
  "questions": ["疑思1"]
}

原文：
{body}
```

---

## 输出格式

### 完整摘录

```
## 📖 《{title}》阅读笔记

📌 **核心观点**
1. {观点1}
2. {观点2}

🔍 **关键证据**
• {证据1}

💬 **金句**
> 「{原文语句}」— {位置}

🚀 **行动项**
- [ ] {行动1}

🤔 **疑思**
• {问题1}

---
📎 来源：{文档链接}
```

### 快速观点

```
## 🎯 《{title}》核心观点

1. {观点1}
2. {观点2}
3. {观点3}

---
> {一句话概括}
```

### 知识卡片

```
---
title: "📇 卡片：{从原文提取的概念名}"
source: "{原文标题}"
tags: [{LLM 提取的标签}]

## 概念
{一句话定义}

## 核心内容
{200-400 字展开论述，从原文提取}

## 关联
- 关联到「{已有概念}」— {关联原因}
  {无已有概念则填「待补充」}

## 思考
{疑思、批判、延伸}

## 来源
{文档链接}
---
```

## 批量模式

```
触发：「批量生成阅读笔记」「逐篇做卡片」「知识库全做摘录」

流程：
  1. yuque_list_docs → 获取文档列表（按 updated_at 降序）
  2. 逐篇 yuque_get_doc 读 body
  3. 逐篇 LLM 生成摘录/卡片
  4. yuque_create_doc 逐篇存入目标库，标题格式：
     摘录：📖 {原标题} — 阅读笔记
     卡片：📇 {原标题}

上限 20 篇，超出提示分批。
输出汇总报告：成果 N/失败 M，目标库链接。
```

---

## 持久化

| 方式 | 触发 | 位置 |
|------|------|------|
| 直接返回 | 默认 | 对话中输出 |
| 存入知识库 | 「存笔记」「保存」「存成文档」 | 默认库或指定库 |
| 批量模式 | 自动 | 逐篇创建文档 |

---

## 参数解析

```
用户输入提取：

  目标文档 → 文档名/slug/doc_id
    doc_id（纯数字）→ yuque_get_doc(book_id=默认库, doc_id)
    文档名 → yuque_search 匹配 → 唯一命中直接用，多个列出选

  模式判定：
    含「卡片/zettelkasten/闪卡」 → 🏷️ 知识卡片
    含「快速/速览/只看观点/要点」 → 🎯 快速观点
    其他 → 📖 完整摘录

  输出方式：
    含「存成文档/保存/写入」 → 存入知识库
    含「批量/全部/所有」 → 批量模式
```

---

## 错误处理

| 场景 | 处理 |
|------|------|
| 文档未找到 | 「未找到文档，检查名称后重试」 |
| 文档内容过短（< 50 字） | 「文档内容过短，不适合生成阅读笔记」 |
| 知识库为空 | 「知识库是空的，没有文档可摘录」 |
| LLM 输出非 JSON | 重试 1 次，仍失败降级为 Markdown 格式 |
| API 限流(429) | 等 Retry-After，最多重试 3 次 |
