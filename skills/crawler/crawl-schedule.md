# yuque_crawl_schedule

> 端点：无（爬虫定时抓取策略分析）
> 说明：与 rss/schedule 共用配置知识库，分析爬虫最近抓取频率，生成推荐下次抓取时间并写回。

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `source` | string | ✅ | 数据源 key，如 `cnblogs` |
| `kv_namespace` | string | ❌ | KV namespace，默认等于 source |
| `mode` | string | ❌ | `analyze`（分析+写回，默认）/ `dry_run`（仅分析不写） |
| `raw` | boolean | ❌ | 返回原始 JSON（默认 false） |

## 频率分档（保守策略）

| 分档 | 条件 | 建议间隔 |
|------|------|----------|
| 高频 | 近 7 天 ≥5 篇 | 每天 1 次 |
| 中频 | 近 7 天 1-4 篇 | 每 7 天 |
| 低频 | 近 14 天 1 篇 | 每 15 天 |
| 休眠 | 近 30 天 0 篇 | 每 30 天 |

## 主方案：配置知识库

与 `yuque_rss_schedule` 共用同一个配置知识库（`rss.schedule.book_id`），文档 slug 前缀 `crawler--`。

文档格式（Markdown）：

```markdown
# 爬虫/cnblogs

- 类型: crawler
- 数据源: cnblogs
- 上次抓取: 2026-06-18
- 推荐下次抓取: 2026-06-19
- 抓取间隔: 1天
- 频率分档: 高频
- 近7天文章数: 15
- 近14天文章数: 28
- 近30天文章数: 45
```

## 使用场景

- 分析爬虫数据源的最近抓取频率，规划定时抓取间隔
- 定期运行（如每天一次），动态更新推荐抓取时间
- 人工查看配置知识库，快速了解各爬虫源状态

## 前置条件

- KV 功能已启用
- KV namespace 中有已抓取的文章记录（v2.7.0+ 格式，含时间戳）
- 主方案需额外配置 `rss.schedule.book_id`

## 注意事项

- 以最近更新情况为准，不看历史总平均
- 数据量越大分析越准确
- 旧格式 KV 数据（v2.6.0 及之前）不含时间戳，会提示重新抓取
- 最小抓取间隔为 1 天
