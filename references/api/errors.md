# 语雀 API 错误码

> 所有 API 端点共用以下错误码。当调用失败时，按状态码判断原因并给出对应处理建议。

## 错误码表

| 状态码 | 含义 | 原因 | 处理 |
|--------|------|------|------|
| 400 | 请求参数非法 | 缺少必填参数、参数类型错误、值超出范围 | 检查参数名和值，对照 API 文档修正后重试 |
| 401 | Token/Scope 未通过鉴权 | Token 无效、过期或权限范围不足 | 检查 `YUQUE_TOKEN` 环境变量，必要时重新生成 Token |
| 403 | 无操作权限 | 已登录但该操作超出当前用户权限 | 确认操作用户是否有该资源的管理/写入权限 |
| 404 | 实体未找到 | 资源不存在或路径错误（login/slug/id 写错） | 检查 namespace、slug、id 是否正确；确认资源未被删除 |
| 422 | 请求参数校验失败 | 参数语义错误（如重复 key、格式不合法） | 检查参数值是否合法（email、URL 格式等），修正后重试 |
| 429 | 访问频率超限 | 每小时 5000 次/每秒 100 次限制 | 重试 1s→2s→3s，最多 3 次。若 `X-RateLimit-Remaining = 0` 则等整点刷新 |
| 500 | 内部错误 | 语雀服务端异常 | 等待片刻重试；若持续报错，联系语雀技术支持 |

## 通用规则

- **401** → 不重试，直接提示用户更新 Token
- **404** → 不重试，确认资源是否存在
- **429** → 重试间隔 1s / 2s / 3s，最多 3 次。若 `X-RateLimit-Remaining = 0` 说明额度耗尽，需等整点刷新（例：14:35 触发则等 25 分钟到 15:00）
- **400/422** → 不重试，检查参数后修正
- **500** → 等待 5 秒后重试，最多 2 次
- **403** → 不重试，确认权限后人工介入
