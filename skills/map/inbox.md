# 碎片收集整理（inbox）

收集小记碎片 → AI 分组聚类 → 整理为结构化主题文档。吃官方 daily-capture。

## 触发词

```
「整理碎片」「收集小记」「整理一下最近的笔记」
「碎片归类」「小记太多该整理了」「定期整理」
「这些碎片能整理成什么」「回顾这周的小记」
```

---

## 三种模式

| 模式 | 输入 | 输出 | 场景 |
|------|------|------|------|
| 📥 收集整理 | 最近 N 条 / 指定时间范围 | 按主题分组的完整文档 | 定期清空小记 |
| 🏷️ 快速归类 | 最近 N 条 +「只归不写」 | 归组建议列表，不生成文档 | 先看看再决定 |
| 📅 定期回顾 | 「回顾这周/月的小记」 | 时间线回顾 + 主题聚类 | 周/月回顾 |

---

## 清理策略（可配置）

用户首次使用或每次整理前询问（如果未明确指定）：

```
整理后的原始小记如何处理？

  A. 全删 — 整理完删除所有原始小记（软删除，可恢复）
  B. 部分删 — 指定保留哪些（如「保留书摘」「保留灵感」）
  C. 不删 — 全部保留，在小记中标注「已整理」
  D. 选着删 — 逐条确认

默认：C（不删）。除非用户说了「清空」「删掉」「归档完清理」。
```

### 清理方式实现

```
选 A → 逐条 yuque_delete_note（软删除，移入回收站）
选 B → 用户指定保留标签/关键词 → 过滤 → 其余删除
选 C → 不调 delete，在整理报告末尾列出「已整理的小记ID」供手动清理
选 D → 逐条展示 → 用户确认 → 逐条删除
```

### 场景适配

```
「整理一下碎片」— 默认不删，等用户后续决定
「清空小记」— 直接选 A，全删
「把书摘整理成文档，其他的删掉」— 选 B，保留书摘
「整理最近的笔记，确认后再删」— 选 D，逐条确认
```

## 核心流程：📥 收集整理

```
1. yuque_list_notes → 获取小记列表
   筛选：最近 N 条、指定时间范围、全部
   上限 30 条，超出提示分批

2. 逐条 yuque_get_note 读完整内容

3. LLM 分组聚类：
   - 分析每条小记的主题和内容类型
   - 同一主题/相关概念归一组
   - 无明确主题 → 归「杂项」
   - 单条小记独占一个主题也保留

4. 每组 LLM 整理为结构化文档：
   - 提取该组共性主题
   - 合并同类观点
   - 删除重复内容
   - 补全缺失的上下文和过渡
   - 标注引用自哪条小记

5. yuque_create_doc 逐组写入知识库

6. 按清理策略处理原始小记

7. 输出汇总报告
```

### 分组策略

```
LLM 按以下优先级分组：

1. 主题相似度 → 内容讨论同一话题 → 一组
2. 上下文关联 → A 提到的概念在 B 中也出现 → 关联
3. 时间连续性 → 连续几天记的同一话题 → 合并
4. 内容类型 → 全部是书摘 / 全部是灵感 / 全部是待办 → 按类型分组

命名规则：
  主题明确 → 📝 {主题名}（{N}条碎片）
  主题模糊 → 📝 杂项整理 {日期范围}（{N}条碎片）
```

### 整理文档格式

```
## 📝 {主题名}

> 整理自 {N} 条小记 | {日期范围}

### 概述
{该主题的总结性描述，LLM 生成}

### 要点
1. {从碎片中提取的要点 1}
2. {要点 2}

### 关键内容
{碎片核心内容展开，保留有用细节}

### 待深入
- {碎片中提到但未展开的点，值得后续研究}
- 无则省略

---
📎 原始小记：{日期} · {日期} · {日期}
```

## 模式二：🏷️ 快速归类

不生成文档，只给分组建议。

```
触发：「只归类不写」「先看看怎么分」「分个类」

输出：

## 🏷️ 碎片归类建议（{N}条）

📌 「{主题名 1}」（{n}条）
  · {小记 1 摘要（≤30字）}
  · {小记 2 摘要}

📌 「{主题名 2}」（{n}条）
  · ...

🔀 无明确主题（{m}条）

---
操作：
  「确认整理」— 按上述分组生成文档
  「排除 主题X」— 跳过改组
  「只整理 主题Y」— 仅处理该组
  「合并 主题A+B」— 两个组合并
  「手动分组」— 用户指定分组
```

## 模式三：📅 定期回顾

```
触发：「回顾这周的小记」「月末回顾」「本周回顾」

流程：
  1. yuque_list_notes 获取全部
  2. LLM 按时间范围筛选（本周/本月）
  3. 按时间线排列 + 按主题聚类

输出：

## 📅 小记回顾：{周/月/日期范围}

### 📊 概览
- 共 {N} 条小记
- 涉及 {M} 个主题
- 最活跃主题：{top 3}

### 🕐 时间线
| 日期 | 主题 | 摘要 |
|------|------|------|
| 2026-05-20 | Docker | 学了 Docker Compose，踩坑记录... |
| 2026-05-21 | Docker | 解决了容器网络问题... |

### 📌 值得展开写成文档的主题
1. {主题} — 已有 {n} 条相关碎片，可以整理成完整文档
2. ...

### 🎯 本周/月收获
{LLM 总结的核心收获，2-3 句话}
```

## 参数解析

```
用户输入提取：

  范围：
    含「最近 N 条」→ list_notes 取前 N 条
    含「这周/本周」→ 筛选 7 天内
    含「这个月/本月」→ 筛选 30 天内
    含「全部/所有」→ 全部小记（上限 30）
    未指定 → 默认最近 10 条

  模式判定：
    含「只归类/只分/先看看」 → 🏷️ 快速归类
    含「回顾」 → 📅 定期回顾
    其他 → 📥 收集整理

  清理策略：
    含「清空/删掉/清理」 → A（全删）
    含「保留 X/除了 X」 → B（部分删）
    含「确认后删/逐条」 → D（选着删）
    未指定 → C（不删，默认）
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 无小记 | 「目前没有小记碎片，尝试用小记功能记录一些想法吧」 |
| 单条小记 | 归类意义不大，建议直接调 write/polish 打磨单条 |
| 超出 30 条 | 「共 {n} 条，首批处理 30 条，剩余可继续」 |
| API 限流(429) | 等 Retry-After，最多重试 3 次 |
| 写入失败 | 单篇失败不影响其他，报告中标注 |
